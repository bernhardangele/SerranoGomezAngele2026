# Articles as flankers: The effect of grammatical gender depends on the task

**Marina Serrano Carot, Pablo Gomez, Bernhard Angele**

This repository contains the data, analysis code, and manuscript source for a series of three experiments investigating whether readers process articles and nouns together, using a novel article flanker paradigm in Spanish.

- **Experiment 1**: Grammatical gender categorization task with article flankers
- **Experiment 2**: Lexical decision task with article flankers
- **Experiment 3**: Semantic categorization task with article flankers

## Repository locations

This repository is stored on both **GitHub** and the **GIN** (G-Node Infrastructure) repository:

| Host | URL |
|------|-----|
| GitHub | <https://github.com/bernhardangele/SerranoGomezAngele2026> |
| GIN | <https://gin.g-node.org/bangele/Serrano_Carot_Gomez_Angele_2026> |

The GitHub repository contains the code and manuscript. The GIN repository additionally contains the large data and model files managed by git-annex.

## Quick start

### Option A: GitHub Codespaces (easiest, no local setup required)

1. Go to the repository on GitHub: <https://github.com/bernhardangele/SerranoGomezAngele2026>
2. Click the green **Code** button, then select the **Codespaces** tab.
3. Click **Create codespace on main**.
4. Wait for the Codespace to build (this may take a few minutes the first time).
5. Once ready, open a terminal in the Codespace (Terminal > New Terminal) and render the manuscript (see [Rendering the manuscript](#rendering-the-manuscript) below).

### Option B: Local Docker (recommended for offline use)

**Prerequisites**: [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.

1. Clone the repository:

   ```bash
   git clone https://github.com/bernhardangele/SerranoGomezAngele2026.git
   cd SerranoGomezAngele2026
   ```

2. Start the container:

   ```bash
   docker compose up
   ```

3. Open RStudio in your browser at <http://localhost:8787> (username: `rstudio`, password: `rstudio`).

4. From the RStudio terminal, render the manuscript (see below).

5. When done, stop the container with `Ctrl+C` in the terminal, then `docker compose down`.

## Getting the data files

The participant data and fitted model files are stored on the GIN repository and managed by **git-annex** via **DataLad**. After cloning, these files appear as symbolic links or small pointer files. To retrieve the actual files, run the provided script from the project root:

```bash
./get_data.sh
```

This script configures DataLad to use the GIN repository over HTTPS (no SSH keys or GIN account required) and downloads all data and model files.

### Retrieving files manually

If you prefer to run the commands yourself:

```bash
# 1. Configure git (required by DataLad)
git config --global user.name "Docker User"
git config --global user.email "docker@example.com"

# 2. Add GIN as a data sibling (HTTPS, no SSH keys needed)
datalad siblings add --name gin --url https://gin.g-node.org/bangele/Serrano_Carot_Gomez_Angele_2026.git

# 3. Download everything
datalad get .

# Or download only specific directories:
datalad get data/
datalad get analysis/*.qs
```

### Re-fitting the models instead of downloading them

The fitted Bayesian model files (`.qs` files in `analysis/`) can also be regenerated by running the R analysis scripts:

```bash
Rscript analysis/analysis_exp1.R
Rscript analysis/analysis_exp2.R
Rscript analysis/analysis_exp3.R
```

**This is not recommended**, especially on Codespaces, because each script fits multiple Bayesian models using `brms`/`cmdstanr` and may take several hours to complete even on a powerful machine. It is much faster to download the pre-fitted models from GIN using `./get_data.sh`.

### Checking if files are already present

After running `get_data.sh` (or `datalad get`), verify that the model files are actual files and not dangling symlinks:

```bash
ls -la analysis/*.qs
```

## Rendering the manuscript

The manuscript is written in **Quarto** using the [apaquarto](https://github.com/wjschne/apaquarto) extension for APA formatting.

From the terminal, run:

```bash
cd final_manuscript
quarto render SerranoGomezAngele2026.qmd
```

This will generate:
- `SerranoGomezAngele2026.pdf` (APA manuscript format)
- `SerranoGomezAngele2026.docx` (Word document)
- `SerranoGomezAngele2026.html` (HTML version)

All R packages, LaTeX dependencies, and the apaquarto extension are already installed in the Docker container.

## Reproducing the analysis from scratch

If you want to re-fit the Bayesian models (this can take several hours even on a powerful machine):

```bash
# From the project root inside the container
Rscript analysis/analysis_exp1.R
Rscript analysis/analysis_exp2.R
Rscript analysis/analysis_exp3.R
```

Each script uses `brms` with `cmdstanr` as the backend. The fitted models are saved as `.qs` files in `analysis/`. The manuscript `.qmd` files load these pre-fitted models when rendering.

## Dependencies

If you prefer not to use Docker, you will need:

- **R** (>= 4.5)
- **Quarto** (>= 1.6) with the [apaquarto](https://github.com/wjschne/apaquarto) extension
- **CmdStan** (2.38.0) and the `cmdstanr` R package
- **DataLad** and **git-annex** (for data retrieval)
- R packages: `brms`, `tidyverse`, `qs2`, `gt`, `flextable`, `ftExtra`, `here`, `papaja`, `conflicted`, and many others (see the [Dockerfile](docker_container/Dockerfile) for the full list)

The Docker image (`bangele1/analysis-in-a-box-rocker:SerranoGomezAngele2026`) has all dependencies pre-installed.
